<h2>COMPARATIVE</h2>
<ul>
<li>
<p>Selenium released in 2004 (opensource; requires java - WebDriver, complex installation because different dependencies version must match: java, selenium, webdriver, protractor; chrome, complex dom manipulation, flaky tests) ~28k stars on Github.</p>
</li>
<li>
<p>Cypress released in 2017 (partially opensource, simple installation, simple dom manipulation, stable tests, but dashboard is closed source, alternative called sorrycypress) ~45k stars on Github.</p>
</li>
<li>
<p>Playwright released in 2020 (opensource, simple installation, simple dom manipulation, stable tests, powerfull tooling) ~57k stars on Github.</p>
</li>
<li>
<p>Others solutions are: WebdriverIO, Puppeteer, Nightwatch, Protractor (NES)</p>
</li>
<li>
<p>The main arguments to use Cypress and Playwright is they are supported by Angular and Nx, that it manages tests directly with the browser; via Chrome DevTools Protocol (CDP) or directly run on it (Cypress) compared to Selenium that uses an intermeriary tool called WebDriver to communicate with the differents browser that makes tests flaky.</p>
</li>
<li>
<p><a href="https://www.rainerhahnekamp.com/en/angular-e2e-testing-protractor-is-dead-long-live-cypress/">https://www.rainerhahnekamp.com/en/angular-e2e-testing-protractor-is-dead-long-live-cypress/</a></p>
</li>
<li>
<p><a href="https://www.lambdatest.com/blog/playwright-vs-selenium-vs-cypress/">https://www.lambdatest.com/blog/playwright-vs-selenium-vs-cypress/</a></p>
</li>
</ul>

async function waitForMessageAsync(): Promise&#x3C;boolean> {
  return new Promise(function (resolve) {
    page.on('console', (msg) => {
      if (msg.type() === 'log' &#x26;&#x26; msg.text() === 'Hello, World!') {
        resolve(true);
      }
    });
  });
}

<h2>CYPRESS: NOTES</h2>
<ul>
<li>the difference between the then() and the within(). then() allows you to do imperative/js on the get element itself while the within() is in the elements inside the get</li>
<li>the difference between the within() and find():  within() allows you to do imperative/js and the find is classic chaining</li>
</ul>
<p>note: you can also use then then() to trigger a function that use the cypress chaining, similar to a command but inside a function</p>
